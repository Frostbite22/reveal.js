4-Github

GitHub, Inc. is a provider of Internet hosting for software development and version control using Git.

It offers the distributed version control and source code management (SCM) functionality of Git, plus its own features. It provides access control and several collaboration features such as bug tracking, feature requests, task management, continuous integration and wikis for every project.

Push your files to remote repository steps

Basic branching

Basic branching

In order to keep your project clean (i.e keep the main branch clean) when collaborating with many contributors, it is highly advisable to work on different branches, branches will allow you to make new pushes and commit and then if merging them on the main branch

merging on the main branch means add the new changes of your new branch to the main branch without making merge conflicts which can happen if we don't follow some guidlines

After creating your branch, you can now add and modify files

the process we have seen above, git add, git commit, git push, enter your credentials

After pushing to the new-branch, suppose you want to make a pull request (which mean merging your new-branch with the main branch), then you have first to check if there aren't any changes in the remote main branch, this happens when you work with many collaborators who also can push to the main branch in this case, for a simple solo project, you don't need this step
keywords : merge, fetch, pull, remote, upstream
now from your github profile, click the button merge pull request, if there are no conflict you'll get a green text, able to merge to the main branch.

This is not everything !

Read also about : fork, rebase, clone, ...

Github account step up (step by step)

  1. Create a Github account
  2. Click on the down arrow near your profile icon on the right
  3. Go to settings -> developer settings -> personal access token -> generate token
  4. Save the token in a text file : this token will allow you to push changes to your repository ; enter the token instead of this password (old since august 2021)
  5. follow the previous steps (add, commit, push) to push files on your repo
